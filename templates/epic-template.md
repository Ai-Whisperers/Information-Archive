# Epic Template

Copy this template when creating new epics.

---

```markdown
## EPIC-XXX: [Title]

**Status:** To Do | In Progress | Done
**Priority:** High | Medium | Low
**Owner:** @Kyrian | @Ivan | @Jonathan
**Phase:** 1 - Foundation | 2 - Optimization | 3 - Innovation

### Problem Statement
[Describe the problem or opportunity this epic addresses]

### Objective
[Clear statement of what success looks like]

### Success Criteria
- [ ] Measurable outcome 1
- [ ] Measurable outcome 2
- [ ] Measurable outcome 3
- [ ] Measurable outcome 4

### Stories
- STORY-XXX: [Title]
- STORY-XXX: [Title]
- STORY-XXX: [Title]

### Related Repositories
- https://github.com/Ai-Whisperers/[repo-name]
- https://github.com/Ai-Whisperers/[repo-name]

### Risks
| Risk | Impact | Mitigation |
|------|--------|------------|
| Risk 1 | High/Medium/Low | Strategy |

### Notes
- Additional context
- Dependencies on other epics
- External factors
```

---

## Field Definitions

| Field | Description | Required |
|-------|-------------|----------|
| ID | Unique identifier (EPIC-001 format) | Yes |
| Title | High-level goal description | Yes |
| Status | To Do/In Progress/Done | Yes |
| Priority | High/Medium/Low | Yes |
| Owner | Primary responsible team member | Yes |
| Phase | Project phase (1-3) | Yes |
| Problem Statement | What problem this solves | Yes |
| Objective | Clear success definition | Yes |
| Success Criteria | Measurable outcomes | Yes |
| Stories | 5-8 stories per Rule 13 | Yes |
| Related Repos | GitHub URLs | Yes |
| Risks | Risk assessment | Recommended |

## Phase Definitions

| Phase | Focus | Epic Limit |
|-------|-------|------------|
| 1 - Foundation | Core setup, consolidation | 3-5 epics |
| 2 - Optimization | Improvement, automation | 3-5 epics |
| 3 - Innovation | New features, experiments | 2-3 epics |

## Epic Limits (Rules 11, 19)

- Maximum 3-5 epics per project phase
- Maximum 10 active epics across all repos
- Each epic should have 5-8 stories (Rule 13)
