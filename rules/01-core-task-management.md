# Core Task Management Rules (1-10)

## Rule 1: Unique Task IDs
Every task must have a unique alphanumeric ID (e.g., TASK-001) for tracking across Ignition views.

## Rule 2: Actionable Descriptions
Task descriptions must be actionable, starting with a verb (e.g., "Implement filter UI").

## Rule 3: Priority Levels
Define priority levels:
- **High**: Urgent or blocks others
- **Medium**: Important
- **Low**: Enhancements

## Rule 4: Effort Estimation
Include estimated effort in hours or days for each task and subtask.

## Rule 5: Status Options
Status options limited to:
- To Do
- In Progress
- Review
- Done
- Blocked
- Outdated

## Rule 6: Repository Linking
Link tasks to specific repositories via Git URL or issue/PR number.

## Rule 7: Explicit Dependencies
Dependencies must be explicitly listed (e.g., "Depends on TASK-005").

## Rule 8: Realistic Deadlines
Set realistic deadlines tied to repo milestones, not exceeding one week for tasks.

## Rule 9: Assignee Acceptance
Require assignee acceptance before moving from To Do to In Progress.

## Rule 10: Task Archival
Archive completed tasks after 30 days to keep dashboards lean.
