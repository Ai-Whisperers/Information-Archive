# Role Assignment Rules (21-30)

## Rule 21: Assignee Pool
Assign only to @Kyrian, @Ivan, or @Jonathan based on expertise.

## Rule 22: Kyrian's Focus
@Kyrian: Outreach, client investigation, non-technical PM (max 20% of tasks).

## Rule 23: Ivan's Focus
@Ivan: Tool development, scripting, automation (e.g., scrapers, analyzers).

## Rule 24: Jonathan's Focus
@Jonathan: Backend, hosting, architecture, integrations.

## Rule 25: Self-Assignment Review
No self-assignment without peer review tag (e.g., @Jonathan-reviewed).

## Rule 26: Load Balancing
Balance weekly load: No assignee exceeds 60% of active tasks.

## Rule 27: Role Filters
Role filters in Ignition dashboard hide irrelevant tasks.

## Rule 28: High-Priority Escalation
Escalate unassigned High-priority tasks to @Jonathan daily.

## Rule 29: Cross-Training
Cross-training: Rotate 10% of tasks outside core role quarterly.

## Rule 30: Assignment Logging
Log assignment changes with reason in task comments.
