# Workflow and Cadence Rules (31-40)

## Rule 31: Daily Standups
Daily standups: 15 mins to update statuses via Ignition or Git.

## Rule 32: Weekly Epic Planning
Weekly epic planning: Generate 2-3 new stories per active repo.

## Rule 33: End-of-Day Sync
End-of-day sync: Pull Git commits/issues into dashboard.

## Rule 34: Blocker Reporting
Blockers must be reported within 4 hours with mitigation plan.

## Rule 35: Review for Done Status
Reviews required for Done status: Peer check by another role.

## Rule 36: Friday Freeze
No status changes on Fridays after 3 PM to avoid weekend gaps.

## Rule 37: Audit Logs
Audit logs capture all updates with timestamps and user IDs.

## Rule 38: Monthly Retrospectives
Monthly retrospectives: Adjust rules based on velocity metrics.

## Rule 39: Inactive Repo Kickstart
Inactive repos get quarterly "kickstart" stories auto-generated.

## Rule 40: Cadence Alerts
Cadence alerts in dashboard for missed deadlines.
